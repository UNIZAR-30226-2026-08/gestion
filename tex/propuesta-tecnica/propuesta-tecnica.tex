\section{Resumen ejecutivo}
El presente documento recoge la propuesta técnica y económica correspondiente
al sistema para jugar a Magnate, un juego de mesa por turnos inspirado en el
popular juego Monopoly, pero con más reglas que esperan hacer de la 
experiencia de juego novedosa y emocionante.
Estas incluyen un tablero en dos niveles, un nuevo sistema de tarjetas,
más formas de desplazamiento a lo largo del tablero y un nuevo sistema de
puntuación.
El sistema a desarrollar consiste en un juego multiplataforma, para escritorio
y web, online y multijugador, con funcionalidades de tipo de red social.
Tendrá un diseño sencillo, estético y lúdico. 

El sistema será compatible con el navegador web Chromium 144.0.7559.96 y
para aplicaciones de escritorio con el sistema operativo Arch Linux 6.18.6.
Se desplegará en varios 
contenedores Docker en un servidor de la empresa 08 -- Roberta Williams.
Las tecnologías a utilizar abarcan Godot 4.6, React, Django y PostgreSQL.

El desarrollo del sistema se realizará a lo largo de los meses de febrero,
marzo, abril y mayo, realizando una primera demostración de una versión con
funcionalidades básicas del sistema el día 10 de abril y presentando el
11 de mayo el código y versión final. El precio del sistema y el proyecto 
suman 30795,50 €. El equipo encargado del proyecto 
consiste en los estudiantes matriculados en 5º curso del Programa Conjunto en 
Matemáticas e Ingeniería Informática. 

En la sección \ref{sec:objetivos} se recoge una descripción en alto nivel
de los requisitos funcionales del sistema, así como una serie de bocetos
y primeros diseños de la interfaz para ilustrar estos.

En la sección \ref{sec:tecnica} se especifican las necesidades técnicas
tanto para los usuarios del sistema como para el cliente, incluidas las
particularidades del despliegue del sistema.

En la sección \ref{sec:plan} se detallan los hitos y fechas relevantes en
la realización del proyecto.

En la sección \ref{sec:equipo} se presentan a los miembros de la empresa
08 -- Roberta Williams encargados del proyecto y la relación de sus cargos.
Además, se aporta un portfolio con proyectos y experiencias similares de los

En la sección \ref{sec:propuesta} se recogen los detalles económicos de

\section{Objetivos del sistema}
\label{sec:objetivos}
El sistema a desarrollar consiste en un juego de inspiración en el Monopoly, 
llamado Magnate. Este juego es multiplataforma
(para escritorio y web), online y multijugador. Contará con funcionalidades de tipo de
red social y otras características atractivas al juego como la inclusión de ``skins'' 
personalizables.

\subsection{Análisis de requisitos preliminar}
En concreto, los requisitos funcionales del sistema incluyen los siguientes:
\begin{enumerate}
\item El sistema permitirá jugar a un juego de inspiración en el Monopoly, que
contenga sus reglas básicas y una combinación de reglas de expansiones
inusuales (como el uso de 3 dados y el párking) y nuevas reglas para el
desplazamiento y la acumulación de efectos. En concreto, las reglas pueden
consultarse en el Apéndice \ref{sec:reglas}.
\item El usuario se puede registrar en el sistema, y de él se guardará 
información relativa a su cuenta de usuario.
\item Se guardará información de las partidas pausadas y en curso. 
\item Se almacenará el historial de las partidas de los usuarios.
\item Las partidas se podrán pausar y reanudar en otra plataforma.
\item El sistema tendrá características de red social, como registrar
		amigos, comparar estadísticas y desafiarse a partidas.
\item El sistema permitirá la creación automática y manual de salas de 
		partidas con otros jugadores.
\item El sistema permitirá un tipo de comunicación simultánea durante la
		partida, como un chat o uso de emoticonos.
\item El sistema permitirá la personalización de la ficha del juego y la
		adquisición de los distintos modelos de estas o ``skins'' disponibles
		mediante un sistema de gacha.
\item El sistema contará con una IA contra la que se podrán jugar partidas de
		forma local.
\end{enumerate}

\section{Descripción técnica}
\label{sec:tecnica}
En esta sección se recogen las necesidades técnicas para emplear la aplicación, tanto por parte
de los usuarios finales como del cliente, y se incluye un esbozo de la tecnología empleada.

\subsection{Necesidades de los usuarios finales}
El sistema será accesible tanto desde navegadores web como ordenadores, pudiendo parar partidas y
reanudarlas desde otro dispositivo. Las tecnologías mínimas compatibles con el sistema, por tipo
de plataforma, son las siguientes:
\begin{itemize}
		\item Navegador web: Chromium 144.0.7559.96.
		\item Aplicación de escritorio compatible con sistemas operativos: Arch Linux 6.18.6.
\end{itemize}

\subsection{Necesidades técnicas del sistema}
El sistema utilizará las siguientes tecnologías:
\begin{itemize}
		\item Godot 4.6 (implementación del sistema para aplicación de escritorio).
		\item React (implementación del sistema para navegador web).
		\item Django (backend).
		\item PostgreSQL (backend de la BDD).
\end{itemize}
Su arquitectura será la de un sistema desplegado en 3 niveles (o 4, según los 
resultados del análisis de prestaciones), utilizando un contenedor para la
base de datos, otro para el backend y la lógica y otro para el frontend (o 2).

La tecnología utilizada para el despliegue serán contenedores Docker.
Se ha tomado esta decisión dada su amplia compatibilidad, tanto para sistemas
Windows como Linux.

El código fuente estará disponible para el cliente. 
Se pondrá a disposición del cliente un archivo tar.gz con los archivos de
configuración Dockercompose para desplegar los contenedores del sistema
en cualquier sistema compatible con Docker. 
Se hará un despliegue en los servidores de nuestra empresa, la compañía 
08 -- Roberta Williams, para la demostración del sistema resultante.
Tras este despliegue inicial, el cliente
podrá decidir continuir con el despliegue en nuestros servidores o
desplegar el sistema en AWS o cualquier plataforma cloud compatible con 
Dockercompose, o con Kubernetes, 
y realizar cualquier extensión adicional que considere a las funcionalidades,
siempre bajo un nuevo presupuesto.

Las necesidades de software y hardware para el despliegue se han intentado
minimizar.
El sistema operativo de los contenedores proporcionados
será Debian o Alpine Linux. Se eligen estos sistemas de código abierto, 
para los que no hay que pagar licencias, como consideración al coste por 
parte del cliente. 
En cuanto al hardware, con una máquina con 4GB de RAM bastará para realizar
una demostración del sistema completo con la carga de una base de datos
mínima. Las necesidades en memoria crecerán conforme se escalen las bases de
datos, recomendando contar para empezar con una máquina con 8GB de RAM.
